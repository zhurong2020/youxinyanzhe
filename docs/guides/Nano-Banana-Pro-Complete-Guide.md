## 二、核心能力详解

### 2.1 "思维"机制 (Thinking Process)

这是 Nano Banana Pro 与 Midjourney、Stable Diffusion 的**最大区别**：

- **机制**：在生成图像前，模型会先进行逻辑拆解和"规划-执行"
- **示例**：输入"画一个由木头拼成的单词'Woodchuck'，由土拨鼠投掷出来"
  - 传统模型：直接生成一堆乱码
  - Pro版：拆解为 主体(土拨鼠) + 动作(投掷) + 物体(单词) + 材质(木头)
- **API 可查看思维链**：设置 `include_thoughts=True`

**启用方式**：
- Gemini 应用：选择"创建图像" → 选择"思考"模型（非"快速"）
- API：设置 `thinking_level` (low, medium, high)

### 2.2 文本渲染 (Text Rendering)

**核心优势**：约 95% 准确率，业界领先

**成功率与字数关系**：
- 1-3 个词：约 75% 成功率
- 4-8 个词：约 40% 成功率
- 9+ 个词：约 15% 成功率

**最佳实践**：
1. 将文本放在**引号**内并明确指定位置
2. 指定字体风格："粗体、白色、无衬线字体"
3. 添加视觉效果："带有厚重白色轮廓和投影"
4. 中文标题保持简短（**4-6字以内成功率最高**）
5. 长文本建议先生成图像再用外部工具添加

**示例提示词**：
```
创建一张海报图像。
顶部用大号粗体无衬线字体显示"醒来吧，梦想家"，
字体为白色，带有 2 像素黑色描边和柔和的投影效果。
文字置于图像顶部三分之一处，居中对齐。
```

### 2.3 Google 搜索接入 (Grounding)

**最具差异化的能力**——模型可连接实时信息：

**适用场景**：
- 教育内容：科学准确的图表、生物插图
- 实时可视化：当前天气信息图、实时数据图表
- 事实性信息图：食谱说明、植物护理指南、历史地图
- 技术文档：准确的技术图表、培训手册

**示例提示词**：
```
可视化主要科技公司的当前股票价值和趋势。
为每家公司添加解释，说明最近发生了什么可能解释该趋势的事件。
风格：干净、现代的数据可视化，颜色编码一致。
```

**重要警告**：对于信息图、图表或数据驱动的视觉输出，**务必验证事实准确性**。

### 2.4 多图融合与角色一致性

**技术规格**：
- 最多 14 张参考图像
- 可维持最多 5 人的身份一致性
- 支持 6 张高保真模式

**核心技术：显式角色分配**——必须明确定义每张图像的角色：

```
使用图像 A 作为角色姿势参考，
使用图像 B 作为艺术风格参考，
使用图像 C 作为背景环境参考。
保持图像 A 中人物的面部特征完全一致。
```

**多图融合模板**：
```
系统意图：创建一张单一合成图像，将所有参考融合为连贯、照片写实的场景。

输入：
- 图像 A（主体）：[描述/上传]
- 图像 B（背景）：[描述/上传]
- 图像 C（风格）：[描述/上传]

层级：
- 身份优先级：图像 A
- 光照权威：图像 B
- 色彩调色板：图像 C

约束：
- 相机角度：[规格]
- 宽高比：[比例]
- 负面：无伪影，无光晕
```

**角色一致性最佳实践**：
- 使用高分辨率参考图像（建议最低 1024×1024）
- 提供多角度、多表情的参考
- 使用清晰、光线良好的图像
- 明确声明身份保持要求
- 在提示词前 10 个词内包含身份标签
- 固定视角："四分之三视角，中景，平视"
- 定义 2-3 个品牌十六进制颜色代码
- 使用一个标志性配饰作为锚点（如"银色星星耳环"）

## 三、提示词框架体系

### 3.1 框架一：Google 官方六要素公式（推荐首选）

| 要素 | 说明 | 示例 |
|------|------|------|
| **主体** | 图像中是谁/什么（要具体） | "一个面带沉思表情的机器人咖啡师，眼部发着蓝光" |
| **构图** | 镜头取景方式 | "特写镜头、广角全景、低角度仰拍、肖像构图" |
| **动作** | 正在发生什么 | "正在冲泡咖啡、施展魔法、奔跑中" |
| **场景** | 场景发生在哪里 | "火星上的未来主义咖啡馆、黄金时刻的阳光草地" |
| **风格** | 整体视觉美学 | "3D 动画、黑色电影、水彩画、照片写实、90年代产品摄影" |
| **编辑指令** | 用于修改图像 | "把领带改成绿色、移除背景中的汽车" |

**实操示例**：
```
创建一张图像：一只穿着小巫师帽的蓬松三花猫，
正在舒适的阳光房里打盹，
阳光从窗户斜射进来形成光斑，
采用温暖的金色调色彩，
风格为吉卜力工作室的手绘动画风格。
```

**适用场景**：日常创意生成、社交媒体内容、快速概念验证

### 3.2 框架二：专业影视模板（摄影/视觉专业人士）

```
[主体 + 形容词] 正在 [动作] 于 [场景/语境]。
[构图/摄影机角度]。
[光照/氛围]。
[风格/媒介]。
[特定约束/文本]。
```

**实操示例**：
```
一个半透明玻璃材质的机器人咖啡师，
正在赛博朋克风格的咖啡馆里拉花，
微距特写镜头，浅景深（f/1.8 光圈），
被霓虹粉和青色招牌的反光照亮，
电影级 8K 渲染，Octane 渲染风格。
机器人胸部显示屏上用粗体 LCD 字体显示"醒来"两字。
```

**适用场景**：产品摄影、电影级概念图、专业广告素材

### 3.3 框架三：提示词画布 (Prompt Canvas) —— 复杂项目首选

这是一个**结构化的 8 区域方法**，专为复杂视觉项目设计：

```
【工作台面】定义视觉格式（仪表板、漫画、蓝图、信息图）

【布局】指定空间排列（网格、列、层级）

【组件】列出必需元素作为逻辑锚点

【风格】定义美学和情绪

【约束】防止错误的规则

【源材料】背景或数据

【意图/目标】图像的用途

【解释】对模糊元素的处理指令
```

**实操示例**：
```
【工作台面】创建一个 3 格漫画页面。

【布局】
- 水平条状布局，各格宽度相等
- 角色位于每格左侧三分之一处
- 对话气泡区域在上方，间距一致

【组件】
- 主角（侦探）
- 城市街道背景
- 雨水和倒影
- 两个对话气泡
- 第三格中有一辆停放的汽车

【风格】1970 年代法国黑色电影漫画美学。柔和色调，粗重线条。

【约束】
- 气泡与面部不重叠
- 文字在小尺寸下必须清晰
- 角色设计在各格中必须保持一致

【源材料】
第一格：角色独自在雨中行走。
第二格：面部特写，陷入沉思。
第三格：汽车在路灯下等待。
```

**适用场景**：多面板漫画、品牌设计系统、复杂信息图、UI 设计

### 3.4 框架四：ICS 极简方法（快速入门）

最简单的三部分结构：

- **I**mage type（图像类型）：蓝图、信息图、图表、海报
- **C**ontent（内容）：源数据或信息
- **S**tyle（风格）：视觉风格（生存指南、麦肯锡演示文稿、漫画）

**实操示例**：
```
图像类型：信息图
内容：光合作用的完整过程，包括阳光、二氧化碳、水、葡萄糖的转化
风格：极简主义科学图解，干净的线条和柔和的绿色调
```

**适用场景**：快速概念验证、教育内容、数据可视化

### 3.5 框架五：推理引导型框架（最高价值）

专为 Gemini 3 的"Thinking"能力量身定制：

**核心结构**：`[角色/身份设定] + [任务目标] + [逻辑推理/布局规划] + [视觉细节描述] + [技术参数约束]`

**实操示例**：
```
你是一位专业的电影概念设计师（角色）。请为一部赛博朋克风格的电影设计一张概念图（任务）。

逻辑规划如下：画面需要展示新旧世界的冲突。前景是破败的贫民窟，代表旧时代的遗留；背景是高耸入云的霓虹科技城，代表新秩序的压迫。两者通过一座巨大的、半透明的全息桥梁连接，寓意着极其有限的阶级流动通道（逻辑推理）。

视觉细节描述：前景的街道潮湿、拥挤，充满了细节，如卖拉面的机器人摊位，灯光是暖色调的灯笼，营造烟火气；背景的城市冷峻、蓝紫色调，建筑表面布满全息广告，显得冷漠而遥远。桥梁上用红色光束显示着'HOPE'字样，但在雾气中显得模糊不清。

技术参数约束：21:9 电影画幅，变形宽银幕镜头效果，光圈 f/2.8 以虚化次要细节，焦距 35mm，ISO 800，保留一定的胶片颗粒感。
```

**适用场景**：复杂场景合成、数据可视化图表、具有强叙事性的画面、需要精确空间关系的构图

## 五、精确性与创造性的平衡策略

### 5.1 分层递进法

不要试图在单个提示词中塞入所有细节。采用"先框架后细节"的策略：

**第一轮（建立基础）**：
```
创建一张现代极简主义客厅的高质量照片，
有灰色沙发、浅木色茶几和大型盆栽。
```

**第二轮（精炼细节）**：
```
把沙发颜色改成深海军蓝。
```

**第三轮（添加元素）**：
```
在茶几上添加三本堆叠的书籍。
```

### 5.2 语义负面提示（而非"不要"指令）

模型对否定指令的理解有限，使用正面描述代替：

❌ **不推荐**：
```
一条繁忙的街道，不要有汽车，不要有行人
```

✅ **推荐**：
```
一条空旷、荒凉的街道，没有任何交通痕迹，
只有偶尔飘过的落叶和斑驳的阳光
```

### 5.3 意图优先于细节

提供上下文意图比堆砌细节更有效：

❌ **标签堆砌（Tag Soup）**——已过时的方法：
```
三明治, 4k, 写实, 专业, 高质量, 美食
```

✅ **意图导向**：
```
为巴西高端美食烹饪书创建一张三明治图像
```

模型会自动推断：专业摆盘、浅景深、完美光照、编辑级质感。

### 5.4 80% 法则——编辑而非重新生成

如果生成的图像已达到 80% 满意度，不要重新生成。使用对话式编辑进行微调：

```
很好，但请把光照改成日落氛围，并让文字变成霓虹蓝色。
```

这不仅节省时间和配额，还能保留已经成功的元素。

### 5.5 约束与创意的黄金比例

根据用户测试，最佳实践是：

- **60% 明确约束**：主体、构图、核心风格
- **40% 留白空间**：让模型发挥其训练知识

**示例**：
```
创建一幅充满活力的都市夜景，
霓虹灯倒映在湿润的柏油路上，
采用赛博朋克色调。
[其余交给模型的创意判断]
```

### 5.6 提示词长度建议

Google 内部研究表明：**25 个词以内的提示词在构图准确率上比冗长段落高 30%**。

## 六、实操案例库

### 案例一：商业产品摄影

```
生成一张高端商业产品摄影图。

主体：一瓶透明的长方形玻璃香水瓶，瓶身正中央印有清晰的衬线体文字'Ocean Breeze'（确保拼写正确）。瓶内液体为淡蓝色。

环境：香水瓶放置在一块潮湿的黑色火山岩上，背景是波涛汹涌的深蓝色大海，激起的白色浪花在空中冻结。

光影：清晨的冷色调自然光，侧逆光照明，在瓶身边缘形成明亮的高光（Rim Light），体现玻璃的通透质感。

细节：画面中有飞溅的水珠，焦距 100mm 微距镜头，极高画质，8k 分辨率。
```

### 案例二：角色一致性分镜

## 八、使用限制与注意事项

### 8.1 配额限制

| 用户类型 | 配额 | 水印 |
|---------|------|------|
| 免费用户 | 每日约 **2-3 张** | Gemini 闪光水印 |
| Google AI Pro | 更高配额 | Gemini 闪光水印 |
| Google AI Ultra | 每日约 **1,000 张** | 仅 SynthID 数字水印（不可见） |

### 8.2 定价（API）

| 分辨率 | 价格 |
|--------|------|
| 1K/2K | 约 $0.067/张 |
| 4K | 约 $0.12-0.15/张 |
| 批处理 | 可享 50% 折扣 |

### 8.3 技术限制

1. **小面孔问题**：复杂场景中的小面孔可能无法准确渲染
2. **拼写错误**：尽管改进显著，偶尔仍会出现拼写错误
3. **事实不准确**：信息图和图表可能误解信息——**必须验证**
4. **多语言问题**：语法、拼写、文化细微差别可能不正确
5. **复杂编辑**：蒙版编辑、日夜转换、多图混合可能产生伪影
6. **角色漂移**：虽然一致性可靠，但并非每次都完美
7. **手部/解剖结构**：仍可能出现错误
8. **宽高比挣扎**：模型在维持宽高比方面仍有困难

### 8.4 安全过滤器

Google 对 Gemini 3 系列实施了严格的安全策略：

**触发拒绝的内容**：
- 涉及儿童的特定场景
- 真实公众人物的未授权肖像
- 暴力/NSFW 内容
- 直接的版权形象（如"米老鼠"）

**对策**：
- 使用"插画风格"而非"照片级写实"
- 将描述抽象化
- 避免具体人物动作、年龄或特定身份

### 8.5 重要提醒

- **SynthID 水印**：所有生成的图像都包含不可见的数字水印
- **知识截止日期**：模型核心知识截止于 2025 年 1 月，但搜索接入可获取更新信息

## 九、专业工作流建议

### 9.1 推荐工作流（98% 成功率）

1. **快速草稿**（30 秒提示词）
2. **首次生成**
3. **精炼提示词**（针对特定修复）
4. **如需要，一次重新生成**
5. **外部工具最终润色**

**总时间**：约 6 分钟获得商业级图像

### 9.2 角色一致性项目工作流

- **第 0 天**：从种子照片生成 18 个候选；选择 4 个锚点
- **第 1-3 天**：每日场景，基础提示词 + 场景切换
- **第 4-7 天**：引入轻微姿势变化，保持面部角度固定
- **结果**：带参考的 98% 一致性率 vs 无参考的 71%

### 9.3 配额优化策略

1. **策略性降级**：构思阶段用 Standard 模型（Fast mode）快速迭代
2. **精准打击**：构图满意后切换到 Pro 模型（Thinking mode）进行最终生成
